# ChangeLog 溫和專業入口 UX 設計

日期：2026-05-04

## 背景

ChangeLog 目前已具備易經占卜、學習、AI 解卦、紀錄與回顧能力，產品核心偏向專業的易經學習與占卦紀錄工具。現有入口雖然功能完整，但第一次使用時會直接跳出完整使用說明，主畫面也要求使用者先理解「數字占、金錢卦、籌策、下卦數、上卦數、動爻數」等概念。

這讓零基礎使用者容易在尚未完成第一次占問前就感到門檻過高；同時，產品又不能為了降低門檻而失去專業者需要的卦象、變爻、解卦法則與紀錄深度。

## 目標

第一版 UX 改善採用「專業核心、溫和入口」方向：

- 保留 ChangeLog 作為專業易經記錄工具的定位。
- 讓零基礎使用者第一次打開時能理解「我可以先問一件事」。
- 把術語與進階起卦方式延後揭露，而不是在第一屏要求理解。
- 將首次說明從長篇完整說明改為短歡迎卡。
- 將詳細說明移入 App 內說明中心，並在流程中提供情境提示。

## 非目標

第一版不做「新手版 / 專業版」雙入口，也不做完整的專業工作台分流。未來可以再考慮專業入口、專業預設模式、客戶/案例管理、進階篩選等功能，但這些不屬於本次設計範圍。

第一版也不重做易經資料內容、AI 解卦模型、紀錄資料結構或廣告/BYOK 商業邏輯。

## 受眾策略

產品對外仍可被理解為「易經占卜、學習、記錄與回顧工具」。但使用者第一次進入時，畫面語言應先面向任務，而不是面向術語。

新手使用者的心理路徑：

1. 我有一件想不清楚的事。
2. 我可以先寫下問題。
3. App 會帶我完成一次起卦。
4. 結果頁會用白話提示我怎麼看。
5. 我可以把理解與後續發展記錄下來。

專業使用者的心理路徑：

1. 我可以直接使用進階起卦方式。
2. 我可以看到本卦、之卦、變爻與朱熹解卦法則。
3. 我可以保存紀錄、補充解讀、追蹤後續驗證。
4. 目前沒有獨立專業入口，但專業功能不可被移除或弱化。

## 首頁主流程

目前首頁以「選擇起卦方式」為主要任務。改版後首頁第一層應改成「先寫下問題」：

- 頁面標題與輸入框聚焦在使用者的疑問。
- 問題輸入框應放在第一個主要操作區。
- 預設起卦方式應是最容易開始的方式。
- 數字占、金錢卦、籌策等選項收進「進階起卦方式」區塊，且預設收合。
- 三不占與須知可保留，但應視覺降噪，避免比主要任務更搶眼。

建議首頁第一屏順序：

1. 品牌與短句：例如「先問一件正在猶豫的事」。
2. 問題輸入框：例如「例：我該不該接受這個合作邀請？」。
3. 主要按鈕：例如「開始一卦」。
4. 次要入口：例如「進階起卦方式」。
5. 輕量須知：例如「問題越具體，日後越容易回顧」。

首頁可保留強烈品牌識別，但不應讓品牌圖像擠壓主流程。建議將目前大型中央 logo 改成更大的低透明度背景圖騰：放在畫面最下層，透明度略低，緩慢旋轉；問題輸入、開始按鈕、進階起卦方式與提示文字則覆蓋在背景 logo 上方。這能保留 ChangeLog 的儀式感與視覺記憶點，同時讓第一任務仍是「寫下問題」。

## 起卦方式揭露

首頁不應一開始要求新手理解所有起卦方式。第一版可採用以下資訊層級：

- 預設：一個最容易開始的起卦流程。
- 進階：預設收合，使用者展開後可選數字占、金錢卦、籌策。
- 說明：每個方式旁有一句短提示，而不是長篇教學。

起卦方式短提示範例：

- 數字占：適合快速起卦，輸入三組直覺想到的正整數。
- 金錢卦：模擬擲硬幣六次，保留儀式感。
- 籌策：適合已有起卦結果，手動輸入六爻數值。

若目前實作必須保留數字占欄位，第一版也應避免直接顯示「下卦數、上卦數、動爻數」作為新手第一眼看到的欄位名稱。可改用較白話的欄位描述，並把原術語放在輔助文字或提示中。

## 首次歡迎卡

首次開啟 App 時，不再顯示完整使用說明。改為短歡迎卡，高度應明顯低於目前 85% 高度的說明底 sheet，避免阻斷感。

短歡迎卡不是完整教學，也不是多步驟 wizard。它只負責三件事：

1. 降低第一次打開 App 的阻斷感，讓使用者知道「先問一件事就可以開始」。
2. 建立產品心智：ChangeLog 會協助起卦、整理卦象，並留下日後可回顧的紀錄。
3. 提供清楚出口：按「開始」進入主流程，或按「查看完整說明」進入說明中心。

建議文案：

「先問一件正在猶豫的事。
ChangeLog 會幫你起卦、整理卦象，並留下日後可回顧的紀錄。
完整使用說明可以隨時在右上角查看。」

建議按鈕：

- 主要按鈕：「開始」
- 次要文字連結：「查看完整說明」

首次導覽應支援三條路徑：

1. 第一次開 App → 短歡迎卡 → 按「開始」→ 完成首次導覽。
2. 第一次開 App → 短歡迎卡 → 查看完整說明 → 看說明中心 → 跳回短歡迎卡，帶使用者體驗真實流程。
3. 第一次開 App → 短歡迎卡 → 查看完整說明 → 看說明中心 → 使用者拒絕跳回短歡迎卡 → 視為首次導覽完成。

因此，使用者按下「開始」後，應關閉歡迎卡並將首次導覽狀態標記為已完成。使用者按下「查看完整說明」時，開啟完整說明中心；離開說明中心後，不直接消失，而是提供一個輕量確認，詢問是否「回到開始畫面，照著真實流程問第一件事」。若使用者接受，回到短歡迎卡；若使用者拒絕，直接完成首次導覽並進入主流程。

## 說明中心

現有完整使用說明不應刪除，而是改成 App 內說明中心。右上角 info 按鈕仍可開啟完整說明。

說明中心內容可保留現有主題，但建議重新組織為更容易掃讀的段落：

- 第一次怎麼開始
- 起卦方式差異
- 如何看本卦、之卦、變爻
- AI 啟發觀測模式
- 如何記錄與回顧

隱私、資料與免責聲明不應在說明中心重貼完整長文。說明中心只需要在獨立區塊提供短說明與「查看隱私政策與資料處理說明」連結，連到現有符合 App Store 與 Google Play 審查需求的隱私政策頁面。

詳細說明是「可被找到」，不是「第一次被迫讀完」。

## 情境提示

第一版應在流程中加入少量情境提示，讓術語在使用者需要時出現。

首頁提示：

- 問題輸入框下方：問題越具體，日後越容易回顧。
- 進階起卦方式：不確定怎麼選時，可以先使用預設方式。

結果頁提示：

- 本卦：目前情境的主象。
- 之卦：有變爻時，象徵事情可能轉向的方向。
- 變爻：這次卦象中特別需要留意的變動位置。
- 朱熹解卦法則：協助判斷應優先閱讀哪些卦爻辭。

紀錄頁提示：

- 自己如何解卦：記下你現在的理解，不必一次寫完整。
- 決定怎麼做：把卦象帶來的提醒轉成可觀察的行動。
- 事後驗證：過一段時間回來記錄實際發展。

## 視覺與語氣

語氣應是「溫和、可信、可開始」，避免「考試感」與「只有熟手才看得懂」。

建議使用：

- 「先問一件事」
- 「開始一卦」
- 「日後回顧」
- 「進階起卦方式」
- 「完整說明」

避免第一屏過度使用：

- 下卦數
- 上卦數
- 動爻數
- 朱熹解卦法則
- 籌策算法
- 本卦 / 之卦作為未解釋的主入口詞

專業術語仍可出現，但應在結果、說明中心或提示中出現，並搭配白話輔助。

## 驗收標準

- 首次開啟不再顯示完整長篇使用說明，而是短歡迎卡。
- 首頁第一個主要任務是輸入問題，不是選起卦方式。
- 使用者不用理解三種起卦方式，也能完成一次起卦。
- 進階起卦方式仍可被熟手找到並使用。
- 完整使用說明仍可從 App 內找到。
- 結果頁或流程中提供本卦、之卦、變爻等術語的白話提示。
- 既有紀錄、AI 解卦、學習頁與資料匯入匯出能力不因本次改版被移除。

## 實作風險

- 若只改歡迎卡但首頁仍以術語為主，新手門檻不會真正下降。
- 若過度簡化首頁，專業使用者可能覺得工具變淺；因此進階起卦方式與完整結果資訊必須保留。
- 若情境提示太多，會變成另一種長說明；每個頁面只放當下必要的短提示。

## 後續可延伸

第一版完成後，可以再評估是否加入：

- 專業入口或專業預設模式。
- 起卦紀錄分類與搜尋。
- 老師使用的案例管理欄位。
- 新手學習路徑或每日一卦。
- 說明中心內的互動式教學。

這些延伸方向應在第一版完成並觀察實際使用回饋後再決定。
